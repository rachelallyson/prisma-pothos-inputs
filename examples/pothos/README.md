# Pothos + Prisma + Generated GraphQL Inputs

This example uses [Pothos](https://pothos-graphql.dev/) with the Prisma plugin. **All scalar, enum, and input types are generated by @rachelallyson/prisma-pothos-inputs**; Pothos only wires the builder, Prisma objects, and resolvers.

Uses **Prisma v7**: the client is generated to `generated/prisma`, the database URL is configured in `prisma.config.ts`, and the app uses the `@prisma/adapter-pg` driver adapter (see `src/db.ts`).

## Setup

1. **Install dependencies**

   ```bash
   npm install
   ```

2. **Generate Prisma client and Pothos/Prisma types**

   ```bash
   npm run prisma:generate
   ```

   Ensure `DATABASE_URL` is set (e.g. in `.env` via `dotenv`).

3. **Generate Pothos types from Prisma schema (scalars, enums, input types)**

   From this directory, run (this builds the generator from the repo root and writes `src/__generated__/pothos-inputs.ts`):

   ```bash
   npm run generate:inputs
   ```

   Or from the **repo root**:

   ```bash
   npx @rachelallyson/prisma-pothos-inputs --schema examples/pothos/prisma/schema.prisma --output-pothos examples/pothos/src/__generated__/pothos-inputs.ts
   ```

4. **Generate TypeScript types from the GraphQL SDL** (for type verification)

   ```bash
   npm run codegen
   ```

5. **Verify TypeScript**

   ```bash
   npm run verify
   ```

   This runs `tsc --noEmit`. It fails if the generated GraphQL input types are not assignable to Prisma’s create inputs (see `src/verify-types.ts`).

## What’s in this example

- **`src/__generated__/pothos-inputs.ts`** – Generated by @rachelallyson/prisma-pothos-inputs. Exports `registerPothosTypes(builder)` (DateTime scalar, SortOrder and Role enums, OrderBy and WhereInput/FindManyArgs/Create/Update input types). Pothos does not define these by hand.
- **`src/schema.ts`** – Calls `registerPothosTypes(builder)`, then only defines Prisma objects (`prismaObject`), query/mutation fields, and resolvers. No manual scalars, enums, or input types.
- **`src/normalise.ts`** – Converts GraphQL input (null for optional) to Prisma input (undefined for optional) so resolvers can pass args into the Prisma client.
- **`src/verify-types.ts`** – Ensures the codegen types from the GraphQL schema are assignable to Prisma’s types after normalisation.

Together, this shows that @rachelallyson/prisma-pothos-inputs generates as much of the schema as possible and Pothos only wires resolvers and Prisma objects.
